â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

The Environmental Hazards UI provides visual representation of flood, wildfire,
and heat risk data integrated into existing property views. Hazard data is
surfaced through:

1. Map overlays (portfolio-level view)
2. Property detail cards (individual property view)
3. Risk dashboard (portfolio analytics)
4. Alerts and notifications (threshold-based)

Design Principles:
- Non-intrusive: Hazard data enhances existing views without cluttering
- Color-coded: Consistent color scheme across all hazard types
- Actionable: Shows financial impact and mitigation options
- Multi-tenant: All views respect RLS tenant isolation

- Filter properties by hazard score range

Backend API design for hazard data:

GET /api/properties/{property_id}/hazards
  Returns: PropertyHazards object for single property

GET /api/properties?include_hazards=true
  Returns: List of properties with embedded hazard data

GET /api/portfolio/hazards/summary
  Returns: Aggregated hazard metrics for entire portfolio

GET /api/hazards/high-risk?threshold=0.6
  Returns: List of high-risk properties (from materialized view)

GET /api/hazards/map/overlay/{layer_type}
  Returns: GeoJSON for map overlay (flood zones, fire zones)

POST /api/hazards/refresh/{property_id}
  Triggers: On-demand hazard reassessment for property

POST /api/hazards/alerts/subscribe
  Body: { property_id, threshold, notification_channels }
  Creates: Alert subscription for property

All endpoints respect RLS tenant isolation via JWT claims.
