The Provenance UI provides visibility into data lineage, trust scores, and
change history at the field level. Users can track how data was sourced,
validate its trustworthiness, and audit all changes over time.

UI Components:
1. Field trust indicators (inline badges)
2. Provenance timeline modal (detailed history)
3. Entity trust dashboard (portfolio-level)
4. Low-trust field alerts
5. Evidence viewer

Design Principles:
- Trust-aware: Visual indicators show trust levels at a glance
- Non-intrusive: Provenance info accessible but doesn't clutter main UI
- Auditable: Complete trail of all data changes
- Actionable: Identify and fix low-trust data
